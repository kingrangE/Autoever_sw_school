# 복습

## 변경 가능과 불가능
- Scala Data / Vector Data(Container)
    - Vector Data 세부 분류
        - Sequence 
            - 한 개씩 넘어가며 진행함 (속도가 느림 O(n))
            - Memory 상대적 절약
        - Key-Value Container
            - 접근 속도 빠름 (Key가 Hash되어 있어 O(1)의 시간)
            - Memory 부담 큼

- CQRS - 읽기 / 쓰기 분리하기 ( 프로그램, DB 모두 분리함)
    - 읽기 -> NoSQL (읽기에 강함)
    - 쓰기 -> RDBMS (안정성 좋음)
    - 이렇게 나누는게 MSA 기능에 따라 분류하기
    - 이걸 위해 Kafka를 배울거임
        - CQRS 구현을 했을 때, 한 쪽 수정하면, 일관성 유지를 위해 다른쪽도 Refresh해줘야함. 그걸 도와주는게 Kafka

- 파이썬에서 사용할 수 있는 속성과 도움말 보는 방법
    - 속성 볼라면 : `dir` 프린트 찍어보기
        ```python
        # EX) int 관련 method 확인
        print(dir(int))
        ```
    - 동작이 궁금하면 -> 메서드 설명이 필요하면 : `help` 찍어보기
        ```python
        # EX) int의 __gt__ 메서드 설명 찾기
        help(int.__gt__)
        ```
    - 가장 정확한 것은 Document
        - Document를 잘 읽어내는 것이 중요하다.
    - 파이썬 라이브러리 -> 영어로 써야함. 심사하는 사람들이 외국인
        - Anyway, 그래서 영어를 잘해야 라이브러리도 이해한다.

# 오늘 수업

## 1교시 - 파이썬

### 비교 연산자

- 숫자와 데이터(문자,리스트 등)에서만 가능하다.
    - `>` : 앞에게 초과 
    - `<` : 뒤에게 초과
    - `>=` : 앞에게 이상
    - `<=` : 뒤에게 이상
    - `==` : 같다.
    - `!=` : 다르다.

- 메서드 비교
    - `__gt__` : greater than
    - `__lt__`  : less than
    - `__gte__` : greater than equal
    - `__lte_` : less than equal
    - `__eq_`  : 같다.
    - `__ne__` : 다르다.

- 비교 작업할 때 아무렇게나 그냥 == 쓰는게 아니다.
    1. 우선 자료형이 같은지 확인 -> 속도 빠름
    2. 개수가 같은지 확인 -> 속도 빠름
    3. 위 두 조건을 만족할 때, 하나씩 비교 -> 상대적 느림
    - 이렇게 효율성을 올림

### 산술 비트 연산자 (이거 진짜 중요함.) : 그패픽 프로그램의 색상 관련 기능, 프로그램의 단축키 그리고 삭제/복사 구현에 사용된다.
- 정수 데이터를 2진수로 변환해서 비트 단위로 연산을 수행한 후, 결과를 10진 정수로 리턴하는 연산자 => `시스템 프로그래밍/데이터 분석에서 많이 사용`
- 종류
    1. `&` : AND 연산
        - 사용 예 : 둘 중 하나만 0이면 0임 -> Program 삭제에 이용 (모든 비트 0이면 지워진 것)
    2. `|` : OR 연산
        - 사용 예 : Program 복사/이동에 이용
    3. `^` : XOR 연산
    4. `~` : 1의 보수 연산
        - 사용 예 : 색 반전
    5. `<<` : 왼쪽 시프트 연산자 ( 한 칸 이동마다 *2 효과 )
    6. `>>` : 오른쪽 시프트 연산자 ( 한 칸 이동마다 /2 효과 )

- 진법 변환
    - 그냥 우리가 아는 2진 변환
    - 신기한거
        - -1 : 1111 1111 (1의 2의 보수)
            - 음수가 없다할 때, 2진수에서 가장 큰 수와 같다.
                - 따라서 무한 재생을 -1 값으로 준다. -> 실제 x비트에서 2^x번 재생이란 의미
                - 모두 탐색해서 찾지 못한 경우, 맨 끝까지 갔다는 의미에서 -1을 준다. (2비트일 때, 0,1,2,3가보고 -1)
                    - ex) 파이썬 find
                        ```python
                        print("Hello".find('e')) # 1
                        print("Hello".find('f')) # -1 (못 찾았다는 의미)
                        ```
        - Command, 단축키 -> 2진수를 사용한다.
            - EX, 왼쪽 마우스 1, 오른쪽 마우스 2, 시프트 4 이런식, 0001,0010,0100 이런식으로 입력에 따라 뭐가 같이 눌렸는지 확인할 수 잇음

- 참고 ) 리눅스 개중요함. 클라우드는 리눅스가 젤 중요
